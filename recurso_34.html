4.2 Niveles DE abstracción DE DATOS: lógico/físico

La arquitectura de tres capas ANSI/SPARC

NIVEL DE VISTAS Vista de Usuario: La vista que los usuarios tienen de la base de datos.
Relevancia: Describe la parte de la base de datos que es relevante para cada usuario.
NIVEL LÓGICO Vista Comunitaria: Vista comunitaria de la base de datos.
Estructura: Describe qué datos están almacenados en la base de datos y las relaciones existentes.
NIVEL FÍSICO Representación Física: Representación física de la base de datos en la computadora.
Almacenamiento: Describe cómo están almacenados los datos en la base de datos.

Independencia de datos

  • Independencia lógica: Permite modificar el esquema conceptual sin afectar vistas ni aplicaciones.
  • Independencia física: Permite cambiar el almacenamiento sin alterar el esquema conceptual.

Los esquemas y las vistas externas

Se destacan los siguientes propósitos:

  • Simplificación: Facilitan el acceso a datos complejos mediante consultas predefinidas.
  • Funcionalidad: Permiten adaptar la información según necesidades específicas de uso.
  • Seguridad: Restringen el acceso mostrando solo los datos relevantes para cada usuario.

El Diccionario de Datos o Catálogo del Sistema

Se identifican tres niveles de diccionario de datos:

  • Diccionario conceptual: registra metadatos a un nivel de abstracción alto (modelos E-R).
  • Diccionario lógico: registra requisitos de datos independientemente de la implementación.
  • Diccionario físico: almacena las estructuras de datos reales; las tablas del sistema de un SGBD relacional constituyen un diccionario de datos físico.
