Tipos de JOIN en consultas SQL

Contexto del caso

Situación

En un sistema académico, se requiere generar distintos reportes a partir de tablas relacionadas como Estudiantes, Cursos y Matrículas. El equipo técnico debe decidir qué tipo de JOIN utilizar según el objetivo del análisis: en algunos casos se necesita visualizar únicamente registros coincidentes entre tablas, mientras que en otros se requiere incluir información completa incluso cuando no existen relaciones. Además, se debe considerar la correcta interpretación de valores NULL cuando no hay correspondencia entre registros. La elección incorrecta podría generar resultados incompletos o inconsistentes, afectando la toma de decisiones institucionales. Es momento de analizar cada situación y seleccionar el tipo de JOIN más adecuado para cada requerimiento. ¡Mucha suerte!

Pregunta 1 de 3
1 ¿Qué tipo de JOIN se debe utilizar si se desea obtener únicamente los registros que tienen coincidencias en ambas tablas?
OPCION A

• OPCION A: Utilizar INNER JOIN para recuperar solo filas con valores coincidentes en ambas tablas.

Clic para ver retroalimentación

Retroalimentación OPCION A: Es la opción más adecuada, ya que INNER JOIN retorna exclusivamente las filas que tienen correspondencia en ambas tablas, garantizando consistencia en los datos relacionados.

OPCION B

• OPCION B: Utilizar LEFT JOIN para incluir también registros sin coincidencia en la tabla derecha.

Clic para ver retroalimentación

Retroalimentación OPCION B: Es una alternativa válida en ciertos análisis; sin embargo, incluye registros sin coincidencia, lo que no cumple el criterio de coincidencia estricta requerido.

Pregunta 2 de 3
2 ¿Qué tipo de JOIN permite obtener todos los registros de una tabla, incluso si no tienen coincidencia en la otra?
OPCION A

• OPCION A: Aplicar LEFT JOIN para incluir todos los registros de la tabla izquierda y valores NULL en caso de no coincidencia.

Clic para ver retroalimentación

Retroalimentación OPCION A: Es la opción más pertinente, ya que LEFT JOIN incluye todos los registros de la tabla principal, mostrando NULL cuando no existen coincidencias en la tabla relacionada.

OPCION B

• OPCION B: Aplicar INNER JOIN para asegurar que todos los registros tengan correspondencia en ambas tablas.

Clic para ver retroalimentación

Retroalimentación OPCION B: Es una opción correcta en escenarios de coincidencia estricta; sin embargo, excluye registros sin correspondencia, limitando la visión completa de los datos.

Pregunta 3 de 3
3 ¿Qué tipo de JOIN se debe utilizar para obtener todas las combinaciones posibles entre dos tablas?
OPCION A

• OPCION A: Utilizar CROSS JOIN para generar el producto cartesiano entre las tablas.

Clic para ver retroalimentación

Retroalimentación OPCION A: Es la opción más adecuada, ya que CROSS JOIN genera todas las combinaciones posibles entre registros de ambas tablas, sin requerir condiciones de coincidencia.

OPCION B

• OPCION B: Utilizar FULL OUTER JOIN para incluir registros coincidentes y no coincidentes de ambas tablas.

Clic para ver retroalimentación

Retroalimentación OPCION B: Es una alternativa válida para combinar datos completos; sin embargo, no genera todas las combinaciones posibles, sino únicamente coincidencias y no coincidencias.

¡Actividad Completada!

La selección del tipo de JOIN determina la calidad y alcance de los resultados obtenidos.

  • - INNER JOIN permite recuperar coincidencias estrictas entre tablas.
  • - LEFT JOIN amplía la consulta incluyendo registros sin correspondencia.
  • - CROSS JOIN genera combinaciones completas entre conjuntos de datos.

El dominio de estos operadores optimiza la integración y análisis de información relacional.
