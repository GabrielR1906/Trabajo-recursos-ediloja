A las 02:00 AM el ingeniero de turno requiere registrar una nueva métrica de latencia por lo que ejecuta un comando para añadir una nueva columna llamada Latencia_ms en la tabla de monitoreo.
Función aplicada:
Retroalimentación: La adición de columnas o modificación de estructuras forma parte del lenguaje de definición de datos, encargado de crear y alterar objetos dentro de la base de datos.
A las 03:15 AM dos analistas intentaron actualizar exactamente el mismo registro de Estado de Puerto de un router al mismo tiempo. El sistema puso en fila una solicitud y procesó la otra primero evitando que la información se corrompiera.
Función aplicada:
Retroalimentación: La gestión de accesos simultáneos evita conflictos entre transacciones concurrentes, garantizando consistencia mediante bloqueos o colas de ejecución.
A las 04:30 AM, hubo un fallo en el servidor principal que provocó un reinicio forzado. Al volver a encender, el sistema demoró en el reinicio, tiempo en el cual deshizo las transacciones a medias permitiendo que los datos queden integros.
Función aplicada:
Retroalimentación: Los mecanismos de recuperación restauran el estado consistente tras fallos, deshaciendo operaciones incompletas y preservando la integridad de la información.
A las 05:00 AM, un pasante intenta visualizar las contraseñas encriptadas de los administradores de red pero el sistema le muestra un mensaje de Permiso Denegado .
Función aplicada:
Retroalimentación: Los sistemas de permisos restringen acciones según privilegios asignados, impidiendo accesos no autorizados a datos sensibles.
A las 06:00 AM, el sistema rechazó la inserción de un registro de falla porque el código del equipo enviado por el sensor no existía en el inventario principal.
Función aplicada:
Retroalimentación: Las reglas de integridad aseguran coherencia entre tablas, evitando referencias inválidas o datos inconsistentes.
Antes de finalizar el turno, el ingeniero extrajo un reporte con todos los incidentes resueltos durante la noche utilizando una sentencia SELECT.
Función aplicada:
Retroalimentación: La consulta de datos mediante SELECT corresponde a operaciones de lectura dentro del lenguaje de manipulación de datos.
Para dejar documentado el cambio de las 02:00 AM, el ingeniero consultó el repositorio del sistema para confirmar que la nueva columna Latencia_ms quedó registrada como un tipo de dato numérico (INT).
Función aplicada:
Retroalimentación: El diccionario de datos almacena información sobre estructuras, tipos y definiciones, permitiendo verificar características de los objetos del sistema.
